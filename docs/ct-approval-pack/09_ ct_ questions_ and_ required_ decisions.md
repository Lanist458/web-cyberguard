# CT Questions & Required Decisions

To proceed as an official Pi Browser security feature, CT guidance is requested on:

1. Integration method:
   - Can Web CyberGuard be embedded as an official module with navigation hooks?
   - Or must it be delivered as a DApp only?

2. Browser hook access:
   - URL load/redirect interception allowed?
   - DOM/script inspection allowed (bounded)?

3. Scanning scope:
   - Access to cache/history/downloads/site storage APIs?
   - Permission to clear site data programmatically?

4. Notifications:
   - OS push notifications allowed?
   - Or in-app only banners?

5. Identity and profile fields:
   - Are requested profile fields permitted?
   - Must they be optional?
   - How to access PN username reliably?

6. Privacy constraints:
   - Required disclaimers, retention requirements, or consent flows?

7. Official DApp registry:
   - Is there an official allowlist/registry of Pi DApps/domains?
   - Can Web CyberGuard integrate with it?

8. Approval path:
   - Steps and criteria to become "Pi Browser Official Security Feature"
